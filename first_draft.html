Problem Statement

Why do we need a smarter IDS?

DDoS Attacks are Growing

Distributed Denial-of-Service attacks flood servers with traffic, causing service disruptions. Cloud environments are a prime target due to their high-value nature.

Traditional IDS Limitations

  • Rule-based systems miss novel attack patterns
  • Cannot learn from temporal flow sequences
  • High false-positive rates with fixed thresholds
  • Fail to distinguish "Suspicious" from normal traffic

Our Goal

Build a deep learning IDS that learns temporal patterns across sequences of network flows to classify traffic as Benign, Attack, or Suspicious.

Approach 2A — Temporal Model

BiLSTM + Soft Attention

Core Idea

A Bidirectional LSTM reads each 30-flow window forward and backward, capturing long-range temporal dependencies in network traffic. A soft attention mechanism then weights each timestep — letting the model focus on the most anomalous flows.

Why Bidirectional?

  • Forward pass: context from past flows
  • Backward pass: context from future flows
  • Combined: 256-dim representation per timestep
  • Better at detecting anomalies embedded in normal traffic

Why Soft Attention?

  • Not all 30 flows are equally informative
  • Attention learns which timesteps matter most
  • A single SYN flood burst in a window of benign flows gets highlighted
  • More interpretable than a collapsed hidden state

Architecture

Input (B, 30, 86)
↓
Linear(86→128) + BatchNorm + ReLU + Dropout
Input projection — stabilises training
↓
BiLSTM(hidden=128, layers=2)
Output: (B, 30, 256)
↓
Soft Attention over 30 timesteps
Weighted sum → (B, 256)
↓
LayerNorm → Linear(256→64) → ReLU → Dropout(0.4)
↓
Linear(64→3) — Attack / Benign / Suspicious

Approach 2B — Temporal Model

1D Residual CNN

Core Idea

Treats each feature as a channel and convolves across the 30 time steps — detecting local temporal patterns like repeated SYN bursts in consecutive flows.

Why Residual Connections?

  • Skip connections prevent gradient vanishing in deep layers
  • Each block learns incremental refinements over the input
  • Enables training deeper networks reliably
  • Combined with intra-block Dropout(0.2) for strong regularisation

Training Configuration

AdamW optimizer LR: 3e-4 Weight decay: 1e-3 CosineAnnealingLR Label smoothing: 0.1 Class-weighted loss Batch: 256

Engineering Decisions

Key Challenges & How We Solved Them

⚠️

Class Imbalance — Suspicious Traffic

Suspicious flows (8.4%) cluster at Dec 18–19. A plain time-split gave 0 Suspicious samples in training. Models learned to ignore the class entirely.

✓ Fix: Build windows over full dataset, then stratify the split at window level — all 3 classes in every split.
📉

Data Leakage Risk

If the scaler or feature selectors see test data during fitting, evaluation results are overly optimistic and not representative of real deployment.

✓ Fix: Fit scaler and selectors on first 70% of chronological rows only; apply transform to all.
🔴

Overfitting

High-dimensional input (318 raw features) caused models to memorise training patterns.

✓ Variance threshold + correlation pruning → 86 features
✓ Dropout 0.4, intra-block Dropout 0.2
✓ Label smoothing 0.1, weight decay 1e-3
✓ CosineAnnealingLR for stable training
🪟

Window Labeling

Majority-vote labeling silently drops the rare Suspicious class since benign flows dominate most windows.

✓ Fix: Use label of the last flow in each window — preserves exact ground truth, keeps Suspicious samples visible.
Other important decisions:
Drop flow_id / src_ip / dst_ip Sort by timestamp first Input projection + BatchNorm Gradient clipping (max=1.0) Class-weighted loss
